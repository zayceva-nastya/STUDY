 <div class="refnamediv">
  <h1 class="refname">file_put_contents</h1>
  <p class="verinfo">(PHP 5, PHP 7)</p><p class="refpurpose"><span class="refname">file_put_contents</span> &mdash; <span class="dc-title">Пишет данные в файл</span></p>

 </div>

 <div class="refsect1 description" id="refsect1-function.file-put-contents-description">
  <h3 class="title">Описание</h3>
  <div class="methodsynopsis dc-description">
   <span class="methodname"><strong>file_put_contents</strong></span>
    ( <span class="methodparam"><span class="type">string</span> <code class="parameter">$filename</code></span>
   , <span class="methodparam"><span class="type"><a href="language.pseudo-types.php#language.types.mixed" class="type mixed">mixed</a></span> <code class="parameter">$data</code></span>
   [, <span class="methodparam"><span class="type">int</span> <code class="parameter">$flags</code><span class="initializer"> = 0</span></span>
   [, <span class="methodparam"><span class="type">resource</span> <code class="parameter">$context</code></span>
  ]] ) : <span class="type">int</span></div>

  <p class="para rdfs-comment">
   Функция идентична последовательным успешным вызовам функций
   <span class="function"><a href="function.fopen.php" class="function">fopen()</a></span>, <span class="function"><a href="function.fwrite.php" class="function">fwrite()</a></span> и
   <span class="function"><a href="function.fclose.php" class="function">fclose()</a></span>.
  </p>
  <p class="para">
   Если <code class="parameter">filename</code> не существует, файл будет
   создан. Иначе, существующий файл будет перезаписан, за исключением
   случая, если указан флаг <strong><code>FILE_APPEND</code></strong>.
  </p>
 </div>


 <div class="refsect1 parameters" id="refsect1-function.file-put-contents-parameters">
  <h3 class="title">Список параметров</h3>
  <p class="para">
   <dl>

    
     <dt>
<code class="parameter">filename</code></dt>

     <dd>

      <p class="para">
       Путь к записываемому файлу.
      </p>
     </dd>

    
    
     <dt>
<code class="parameter">data</code></dt>

     <dd>

      <p class="para">
       Записываемые данные. Может быть типа <span class="type"><a href="language.types.string.php" class="type string">string</a></span>,
       <span class="type"><a href="language.types.array.php" class="type array">array</a></span> или ресурсом потока.
      </p>
      <p class="para">
       Если <code class="parameter">data</code> является потоковым ресурсом
       (<span class="type"><span class="type stream">stream</span></span>), оставшийся буфер этого потока будет скопирован 
       в указанный файл. Это похоже на использование функции <span class="function"><a href="function.stream-copy-to-stream.php" class="function">stream_copy_to_stream()</a></span>.
      </p>
      <p class="para">
       Также вы можете передать одномерный массив в качестве параметра
       <code class="parameter">data</code>. Это будет эквивалентно вызову
       <em>file_put_contents($filename, implode(&#039;&#039;, $array))</em>.
      </p>
     </dd>

    
    
     <dt>
<code class="parameter">flags</code></dt>

     <dd>

      <p class="para">
       Значением параметра <code class="parameter">flags</code> может быть
       любая комбинация следующих флагов, соединенных бинарным
       оператором ИЛИ (<em>|</em>).
      </p>
      <p class="para">
       <table class="doctable table">
        <caption><strong>Доступные флаги</strong></caption>
        
         <thead>
          <tr>
           <th>Флаг</th>
           <th>Описание</th>
          </tr>

         </thead>

         <tbody class="tbody">
          <tr>
           <td>
            <strong><code>FILE_USE_INCLUDE_PATH</code></strong>
           </td>
           <td>
            Ищет <code class="parameter">filename</code> в подключаемых директориях.
            Подробнее смотрите директиву <a href="ini.core.php#ini.include-path" class="link">include_path</a>.
           </td>
          </tr>

          <tr>
           <td>
            <strong><code>FILE_APPEND</code></strong>
           </td>
           <td>
            Если файл <code class="parameter">filename</code> уже существует, данные
            будут дописаны в конец файла вместо того, чтобы его перезаписать.
           </td>
          </tr>

          <tr>
           <td>
            <strong><code>LOCK_EX</code></strong>
           </td>
           <td>
            Получить эксклюзивную блокировку на файл на время записи. Другими словами,
            между вызовами <span class="function"><a href="function.fopen.php" class="function">fopen()</a></span> и <span class="function"><a href="function.fwrite.php" class="function">fwrite()</a></span> произойдет
            вызов функции <span class="function"><a href="function.flock.php" class="function">flock()</a></span>. Это не одно и то же, что  вызов
            <span class="function"><a href="function.fopen.php" class="function">fopen()</a></span> с флагом &quot;x&quot;.
           </td>
          </tr>

         </tbody>
        
       </table>

      </p>
     </dd>

    
    
     <dt>
<code class="parameter">context</code></dt>

     <dd>

      <p class="para">
       Корректный ресурс контекста, созданный с помощью функции
       <span class="function"><a href="function.stream-context-create.php" class="function">stream_context_create()</a></span>.
      </p>
     </dd>

    
   </dl>

  </p>
 </div>


 <div class="refsect1 returnvalues" id="refsect1-function.file-put-contents-returnvalues">
  <h3 class="title">Возвращаемые значения</h3>
  <p class="para">
   Функция возвращает количество записанных байт в файл, или
   <strong><code>FALSE</code></strong> в случае ошибки.
  </p>
  <div class="warning"><strong class="warning">Внимание</strong><p class="simpara">Эта функция может возвращать как логическое значение <strong><code>FALSE</code></strong>, так и значение не типа boolean, которое приводится к <strong><code>FALSE</code></strong>. За более подробной информацией обратитесь к разделу <a href="language.types.boolean.php" class="link">Булев тип</a>. Используйте <a href="language.operators.comparison.php" class="link">оператор ===</a> для проверки значения, возвращаемого этой функцией.</p></div>
 </div>

Here is a stupid pitfall I just fell into.<br />I think it may happen rather frequently, so I report it.<br /><br />A common situation is that the $filename argument is built from two variables:<br />file_put_contents($path . $file, $content);<br /><br />Say that $path = 'path/to' and $file = 'file.txt': you see that $path lacks its ending "/", so the resulting full path is 'path/tofile.txt'.<br /><br />Then you look at 'path/to' and don't see any 'file.txt', although no warning or notice was thrown!<br />And may be (like for me :D) it'll take time before you realize that a bad 'tofile.txt' was created in the *parent* directory of the one where you were looking for your file.</span>
